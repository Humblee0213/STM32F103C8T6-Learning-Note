# 04 — OLED显示屏：软件 I²C 驱动、字符显示与数字显示

> 资料说明：当前环境无法直接读取该视频的完整字幕和画面。本笔记依据该课程第 10 个分 P 的主题“[4-2] OLED显示屏”、前后课程上下文以及 SSD1306 OLED 的常见驱动流程整理。文中的核心代码为基于 STM32F103C8T6 标准外设库的软件 I²C 参考实现，不宣称逐字还原视频源码。

## 实验概述

本实验基于 STM32F103C8T6 标准外设库，使用两个普通 GPIO 模拟 I²C 时序，驱动常见的 0.96 英寸、128 × 64 分辨率 OLED 显示屏。

本节主要实现以下功能：

1. 初始化 OLED 显示屏
2. 向 OLED 写入命令和显示数据
3. 设置显示光标位置
4. 清空 OLED 显存
5. 显示英文字符和字符串
6. 显示无符号十进制数
7. 显示带符号十进制数
8. 显示十六进制数和二进制数

本节的核心目标是理解 OLED 驱动程序的分层结构：

> GPIO 模拟 I²C → 发送 OLED 命令/数据 → 设置页地址和列地址 → 查询字模 → 将字模数据写入显存

## 硬件连接

以下接线以常见四针 I²C OLED 模块和软件 I²C 参考实现为例。

| 器件/信号 | 对应引脚 | 说明 |
|---|---:|---|
| OLED VCC | 3.3V | 优先使用 3.3V 供电，具体以模块标注为准 |
| OLED GND | GND | OLED 与 STM32 必须共地 |
| OLED SCL | PB8 | 软件 I²C 时钟线 |
| OLED SDA | PB9 | 软件 I²C 数据线 |
| ST-Link SWDIO | PA13 | 程序下载与调试 |
| ST-Link SWCLK | PA14 | 程序下载与调试 |
| ST-Link GND | GND | 调试器与开发板共地 |
| ST-Link 3.3V | 3.3V | 可作为目标板电平参考或供电，取决于接线方式 |

> 注意：部分 OLED 模块已经在 SCL 和 SDA 上集成上拉电阻；如果模块没有上拉电阻，应外接合适的上拉电阻。软件 I²C 引脚建议配置为开漏输出。

> 地址说明：常见 SSD1306 OLED 的 7 位 I²C 地址为 `0x3C`，发送写操作时，线上实际发送的 8 位地址字节通常为 `0x78`。少数模块可能使用 `0x3D`，应以模块电路和实际扫描结果为准。

### 3. OLED.c：GPIO 与软件 I²C 底层驱动

```c
#include "OLED.h"                                            // 引入 OLED 对外接口声明和 STM32 标准外设库定义
#include "OLED_Font.h"                                       // 引入 8×16 ASCII 字模数组声明
#include "Delay.h"                                           // 引入毫秒延时函数，用于 OLED 上电稳定等待

#define OLED_SCL_PORT GPIOB                                  // 定义软件 I²C 时钟线所在的 GPIO 端口
#define OLED_SCL_PIN GPIO_Pin_8                              // 定义软件 I²C 时钟线使用 PB8
#define OLED_SDA_PORT GPIOB                                  // 定义软件 I²C 数据线所在的 GPIO 端口
#define OLED_SDA_PIN GPIO_Pin_9                              // 定义软件 I²C 数据线使用 PB9
#define OLED_ADDRESS 0x78                                    // 定义 OLED 的 8 位写地址，等于 7 位地址 0x3C 左移一位

#define OLED_W_SCL(x) GPIO_WriteBit(OLED_SCL_PORT, OLED_SCL_PIN, (BitAction)(x)) // 将参数 x 写入 SCL 引脚，统一封装时钟线控制
#define OLED_W_SDA(x) GPIO_WriteBit(OLED_SDA_PORT, OLED_SDA_PIN, (BitAction)(x)) // 将参数 x 写入 SDA 引脚，统一封装数据线控制

static void OLED_I2C_Delay(void)                             // 定义仅供本文件使用的软件 I²C 短延时函数
{                                                            // 软件 I²C 短延时函数体开始
    volatile uint8_t i;                                      // 定义易失循环变量，防止编译器把延时循环完全优化掉
    for (i = 0; i < 10; i++)                                // 执行若干次空操作，为 SCL 和 SDA 电平变化留出稳定时间
    {                                                        // 延时循环体开始
        __NOP();                                             // 执行一条空指令，不改变程序状态但消耗一个 CPU 周期
    }                                                        // 延时循环体结束
}                                                            // 软件 I²C 短延时函数结束

static void OLED_I2C_Init(void)                              // 定义软件 I²C GPIO 初始化函数，仅在 OLED.c 内部调用
{                                                            // 软件 I²C GPIO 初始化函数体开始
    GPIO_InitTypeDef GPIO_InitStructure;                     // 定义 GPIO 初始化结构体，用于配置 SCL 和 SDA 引脚

    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE);    // 开启 GPIOB 外设时钟，否则 PB8 和 PB9 无法正常工作

    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_Out_OD;         // 将 SCL 和 SDA 配置为开漏输出，符合 I²C 总线电气特性
    GPIO_InitStructure.GPIO_Pin = OLED_SCL_PIN | OLED_SDA_PIN; // 同时选择 PB8 和 PB9，保证两根总线一次完成初始化
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;        // 设置 GPIO 最大翻转速度为 50MHz，此参数不等于实际 I²C 频率
    GPIO_Init(GPIOB, &GPIO_InitStructure);                   // 把初始化参数写入 GPIOB 配置寄存器

    OLED_W_SCL(1);                                          // 释放 SCL，使其通过上拉电阻回到高电平空闲状态
    OLED_W_SDA(1);                                          // 释放 SDA，使其通过上拉电阻回到高电平空闲状态
}                                                            // 软件 I²C GPIO 初始化函数结束

static void OLED_I2C_Start(void)                             // 定义软件 I²C 起始条件函数
{                                                            // 软件 I²C 起始条件函数体开始
    OLED_W_SDA(1);                                          // 先释放 SDA，确保起始条件前数据线处于高电平
    OLED_W_SCL(1);                                          // 再释放 SCL，确保起始条件发生在时钟线为高电平期间
    OLED_I2C_Delay();                                       // 等待总线电平稳定，满足起始条件建立时间
    OLED_W_SDA(0);                                          // 在 SCL 保持高电平时把 SDA 从高拉低，形成 I²C 起始条件
    OLED_I2C_Delay();                                       // 保持起始条件一小段时间，保证从机能够可靠识别
    OLED_W_SCL(0);                                          // 拉低 SCL，进入后续数据位传输阶段
    OLED_I2C_Delay();                                       // 等待时钟线低电平稳定
}                                                            // 软件 I²C 起始条件函数结束

static void OLED_I2C_Stop(void)                              // 定义软件 I²C 停止条件函数
{                                                            // 软件 I²C 停止条件函数体开始
    OLED_W_SDA(0);                                          // 先保持 SDA 为低电平，为产生低到高跳变做准备
    OLED_W_SCL(1);                                          // 释放 SCL，使时钟线先回到高电平
    OLED_I2C_Delay();                                       // 等待时钟线高电平稳定
    OLED_W_SDA(1);                                          // 在 SCL 为高电平时释放 SDA，使其由低变高形成停止条件
    OLED_I2C_Delay();                                       // 保持停止条件并等待总线回到空闲状态
}                                                            // 软件 I²C 停止条件函数结束

static void OLED_I2C_SendByte(uint8_t Byte)                  // 定义软件 I²C 单字节发送函数，按照高位在前发送 8 位数据
{                                                            // 软件 I²C 单字节发送函数体开始
    uint8_t i;                                               // 定义循环变量，用于依次发送一个字节中的 8 个数据位

    for (i = 0; i < 8; i++)                                // 循环 8 次，每次发送 Byte 中的一位
    {                                                        // 数据位发送循环体开始
        OLED_W_SDA((Byte & (0x80 >> i)) != 0);              // 取出当前待发送位，并根据位值决定释放或拉低 SDA
        OLED_I2C_Delay();                                   // 等待 SDA 数据电平稳定，满足数据建立时间
        OLED_W_SCL(1);                                      // 释放 SCL 产生高电平，从机在该阶段采样当前 SDA 数据位
        OLED_I2C_Delay();                                   // 保持 SCL 高电平一小段时间，满足有效时钟宽度
        OLED_W_SCL(0);                                      // 拉低 SCL，结束当前数据位并准备下一位
        OLED_I2C_Delay();                                   // 等待 SCL 低电平稳定后再改变下一位 SDA 数据
    }                                                        // 数据位发送循环体结束

    OLED_W_SDA(1);                                          // 发送完 8 位后释放 SDA，为从机返回应答位预留总线控制权
    OLED_I2C_Delay();                                       // 等待 SDA 释放状态稳定
    OLED_W_SCL(1);                                          // 产生第 9 个时钟脉冲，对应 I²C 应答位时钟
    OLED_I2C_Delay();                                       // 保持应答时钟高电平；本简化驱动不读取 ACK 电平
    OLED_W_SCL(0);                                          // 拉低 SCL，结束应答位时钟
    OLED_I2C_Delay();                                       // 等待时钟线低电平稳定
}                                                            // 软件 I²C 单字节发送函数结束

static void OLED_WriteCommand(uint8_t Command)               // 定义 OLED 命令写入函数，用于配置 SSD1306 控制器
{                                                            // OLED 命令写入函数体开始
    OLED_I2C_Start();                                       // 产生 I²C 起始条件，开始一次新的总线事务
    OLED_I2C_SendByte(OLED_ADDRESS);                         // 发送 OLED 写地址，选择地址为 0x3C 的目标设备
    OLED_I2C_SendByte(0x00);                                // 发送控制字节 0x00，表示后续字节应被解释为命令
    OLED_I2C_SendByte(Command);                              // 发送具体控制命令，例如开屏、设置页地址或对比度
    OLED_I2C_Stop();                                        // 产生 I²C 停止条件，结束本次命令写入事务
}                                                            // OLED 命令写入函数结束

static void OLED_WriteData(uint8_t Data)                     // 定义 OLED 数据写入函数，用于向图形显示 RAM 写入像素数据
{                                                            // OLED 数据写入函数体开始
    OLED_I2C_Start();                                       // 产生 I²C 起始条件，开始一次新的总线事务
    OLED_I2C_SendByte(OLED_ADDRESS);                         // 发送 OLED 写地址，选中目标显示模块
    OLED_I2C_SendByte(0x40);                                // 发送控制字节 0x40，表示后续字节应写入显示数据 RAM
    OLED_I2C_SendByte(Data);                                 // 发送一个显示数据字节，其中每一位对应当前页中的一个垂直像素
    OLED_I2C_Stop();                                        // 产生 I²C 停止条件，结束本次显示数据写入事务
}                                                            // OLED 数据写入函数结束
```

### 4. OLED.c：光标、清屏与字符显示

```c
static void OLED_SetCursor(uint8_t Page, uint8_t X)           // 定义 OLED 光标定位函数，Page 表示页地址，X 表示水平列地址
{                                                            // OLED 光标定位函数体开始
    OLED_WriteCommand(0xB0 | Page);                          // 设置页起始地址，每页对应垂直方向连续的 8 个像素
    OLED_WriteCommand(0x10 | ((X & 0xF0) >> 4));             // 设置列地址高 4 位，定位到 0～127 列中的目标位置
    OLED_WriteCommand(0x00 | (X & 0x0F));                    // 设置列地址低 4 位，与高 4 位共同组成完整列地址
}                                                            // OLED 光标定位函数结束

void OLED_Clear(void)                                        // 定义 OLED 清屏函数，向 8 个页的所有列写入 0
{                                                            // OLED 清屏函数体开始
    uint8_t Page;                                            // 定义页循环变量，SSD1306 的 64 行像素被划分为 8 页
    uint8_t X;                                               // 定义列循环变量，用于遍历每页的 128 个显示列

    for (Page = 0; Page < 8; Page++)                         // 依次处理第 0 页到第 7 页，覆盖整个垂直显示区域
    {                                                        // 页循环体开始
        OLED_SetCursor(Page, 0);                             // 把当前写入位置移动到该页的第 0 列
        for (X = 0; X < 128; X++)                            // 遍历当前页的 128 个列地址
        {                                                    // 列循环体开始
            OLED_WriteData(0x00);                            // 写入全 0 数据，使当前 8 个垂直像素全部熄灭
        }                                                    // 列循环体结束
    }                                                        // 页循环体结束
}                                                            // OLED 清屏函数结束

void OLED_ShowChar(uint8_t Line, uint8_t Column, char Char)  // 定义单字符显示函数，使用 1～4 行和 1～16 列的字符坐标
{                                                            // 单字符显示函数体开始
    uint8_t i;                                               // 定义字模列循环变量，用于写入字符的 8 个水平列
    uint8_t Index;                                           // 定义字模索引变量，用于定位字符在字模数组中的位置

    if (Char < ' ' || Char > '~')                            // 判断字符是否超出可打印 ASCII 字符范围
    {                                                        // 非法字符处理分支开始
        Char = ' ';                                          // 将无法显示的字符替换为空格，避免访问字模数组越界
    }                                                        // 非法字符处理分支结束

    Index = (uint8_t)(Char - ' ');                           // 将 ASCII 码转换为从空格开始的字模数组下标

    OLED_SetCursor((uint8_t)((Line - 1) * 2), (uint8_t)((Column - 1) * 8)); // 定位到字符上半部分对应的页地址和列地址
    for (i = 0; i < 8; i++)                                 // 依次写入字符上半部分的 8 个字模字节
    {                                                        // 字符上半部分写入循环体开始
        OLED_WriteData(OLED_F8x16[Index][i]);                // 把当前字符上半部分的一列像素数据写入 OLED 显存
    }                                                        // 字符上半部分写入循环体结束

    OLED_SetCursor((uint8_t)((Line - 1) * 2 + 1), (uint8_t)((Column - 1) * 8)); // 定位到字符下半部分对应的下一页同一列
    for (i = 0; i < 8; i++)                                 // 依次写入字符下半部分的 8 个字模字节
    {                                                        // 字符下半部分写入循环体开始
        OLED_WriteData(OLED_F8x16[Index][i + 8]);            // 把当前字符下半部分的一列像素数据写入 OLED 显存
    }                                                        // 字符下半部分写入循环体结束
}                                                            // 单字符显示函数结束

void OLED_ShowString(uint8_t Line, uint8_t Column, char *String) // 定义字符串显示函数，从指定字符坐标开始连续输出
{                                                            // 字符串显示函数体开始
    uint8_t i;                                               // 定义字符串索引变量，用于逐个访问字符串中的字符

    for (i = 0; String[i] != '\0'; i++)                      // 从字符串首字符开始遍历，直到遇到 C 字符串结束符
    {                                                        // 字符串遍历循环体开始
        OLED_ShowChar(Line, (uint8_t)(Column + i), String[i]); // 调用单字符函数，把当前字符显示在向右递增的列位置
    }                                                        // 字符串遍历循环体结束
}                                                            // 字符串显示函数结束
```

### 5. OLED.c：数字显示函数

```c
static uint32_t OLED_Pow(uint32_t X, uint32_t Y)              // 定义整数幂运算辅助函数，用于分离十进制、十六进制或二进制位
{                                                            // 整数幂运算辅助函数体开始
    uint32_t Result = 1;                                     // 定义结果变量并初始化为 1，对应任何数的 0 次方

    while (Y != 0)                                           // 当指数 Y 仍大于 0 时继续执行乘法
    {                                                        // 幂运算循环体开始
        Result *= X;                                         // 将当前结果乘以底数 X，逐步计算 X 的 Y 次方
        Y--;                                                 // 指数减 1，记录已经完成一次乘法
    }                                                        // 幂运算循环体结束

    return Result;                                           // 返回计算得到的整数幂结果
}                                                            // 整数幂运算辅助函数结束

void OLED_ShowNum(uint8_t Line, uint8_t Column, uint32_t Number, uint8_t Length) // 定义无符号十进制数显示函数
{                                                            // 无符号十进制数显示函数体开始
    uint8_t i;                                               // 定义位循环变量，用于从最高位到最低位依次显示

    for (i = 0; i < Length; i++)                             // 按用户指定的 Length 位数执行循环
    {                                                        // 十进制数字显示循环体开始
        OLED_ShowChar(Line, (uint8_t)(Column + i), (char)(Number / OLED_Pow(10, Length - i - 1) % 10 + '0')); // 提取当前十进制位并转换为对应 ASCII 字符显示
    }                                                        // 十进制数字显示循环体结束
}                                                            // 无符号十进制数显示函数结束

void OLED_ShowSignedNum(uint8_t Line, uint8_t Column, int32_t Number, uint8_t Length) // 定义带符号十进制数显示函数
{                                                            // 带符号十进制数显示函数体开始
    uint32_t Magnitude;                                      // 定义无符号幅值变量，用于保存去除正负号后的绝对值

    if (Number >= 0)                                         // 判断待显示数是否为非负数
    {                                                        // 非负数处理分支开始
        OLED_ShowChar(Line, Column, '+');                    // 在首列显示正号，明确表示该数为正数或零
        Magnitude = (uint32_t)Number;                        // 将非负数直接转换为无符号幅值
    }                                                        // 非负数处理分支结束
    else                                                     // 当待显示数小于 0 时进入负数处理分支
    {                                                        // 负数处理分支开始
        OLED_ShowChar(Line, Column, '-');                    // 在首列显示负号
        Magnitude = (uint32_t)(-(Number + 1)) + 1U;          // 安全计算负数绝对值，避免最小负数直接取反时溢出
    }                                                        // 负数处理分支结束

    OLED_ShowNum(Line, (uint8_t)(Column + 1), Magnitude, Length); // 在符号右侧调用无符号显示函数输出数值部分
}                                                            // 带符号十进制数显示函数结束

void OLED_ShowHexNum(uint8_t Line, uint8_t Column, uint32_t Number, uint8_t Length) // 定义十六进制数显示函数
{                                                            // 十六进制数显示函数体开始
    uint8_t i;                                               // 定义位循环变量，用于逐位输出十六进制数字
    uint8_t SingleNumber;                                    // 定义当前十六进制位变量，范围为 0～15

    for (i = 0; i < Length; i++)                             // 按指定十六进制位数从高位到低位循环
    {                                                        // 十六进制数字显示循环体开始
        SingleNumber = (uint8_t)(Number / OLED_Pow(16, Length - i - 1) % 16); // 提取当前十六进制位的数值
        if (SingleNumber < 10)                               // 判断当前位是否属于数字 0～9
        {                                                    // 数字字符处理分支开始
            OLED_ShowChar(Line, (uint8_t)(Column + i), (char)(SingleNumber + '0')); // 把 0～9 转换为 ASCII 数字字符并显示
        }                                                    // 数字字符处理分支结束
        else                                                 // 当当前位为 10～15 时进入字母处理分支
        {                                                    // 十六进制字母处理分支开始
            OLED_ShowChar(Line, (uint8_t)(Column + i), (char)(SingleNumber - 10 + 'A')); // 把 10～15 转换为 A～F 并显示
        }                                                    // 十六进制字母处理分支结束
    }                                                        // 十六进制数字显示循环体结束
}                                                            // 十六进制数显示函数结束

void OLED_ShowBinNum(uint8_t Line, uint8_t Column, uint32_t Number, uint8_t Length) // 定义二进制数显示函数
{                                                            // 二进制数显示函数体开始
    uint8_t i;                                               // 定义位循环变量，用于从最高二进制位依次显示

    for (i = 0; i < Length; i++)                             // 按指定二进制位数执行循环
    {                                                        // 二进制数字显示循环体开始
        OLED_ShowChar(Line, (uint8_t)(Column + i), (char)(Number / OLED_Pow(2, Length - i - 1) % 2 + '0')); // 提取当前二进制位并转换为字符 0 或 1
    }                                                        // 二进制数字显示循环体结束
}                                                            // 二进制数显示函数结束
```

## 关键知识点

### 1. OLED 显示屏的基本原理

#### 原理

OLED 是 Organic Light-Emitting Diode，即有机发光二极管显示器。OLED 像素能够主动发光，不需要像普通 LCD 那样依赖背光源。

常见的 0.96 英寸单色 OLED 模块通常具有：

- 128 × 64 像素分辨率
- SSD1306 或兼容显示控制器
- I²C 或 SPI 通信接口
- 内部图形显示 RAM
- 每个像素只有亮和灭两种状态

STM32 并不是直接逐个驱动 OLED 面板中的像素，而是把命令和显示数据发送给 SSD1306 控制器，再由控制器完成像素扫描和点亮。

#### 特点

- 自发光，对比度高。
- 视角较大。
- 单色 OLED 驱动相对简单。
- 适合作为单片机调试和数据显示工具。
- 无需刷新整个屏幕也能修改局部内容。
- 长时间显示静止图案可能产生残影或老化。

#### 面试易问

**Q：OLED 和 LCD 的主要区别是什么？**

A：OLED 像素可以主动发光，通常不需要背光；LCD 像素本身不发光，需要背光透过液晶层形成画面。

**Q：STM32 是直接控制 OLED 像素吗？**

A：通常不是。STM32 通过 I²C 或 SPI 向 SSD1306 等显示控制器发送命令和数据，由显示控制器管理显存并扫描 OLED 面板。

#### 易错点

- 把 OLED 模块的供电电压和 I/O 逻辑电平混为一谈。
- 忽略模块控制芯片型号，直接套用不兼容的初始化命令。
- 接线没有共地，导致通信完全失败。
- 长时间让高亮静止图案保持不变，增加局部老化风险。

---

### 2. SSD1306 显存、页地址与像素映射

#### 原理

128 × 64 单色 OLED 一共有：

```text
128 × 64 = 8192 个像素
```

每个像素只需要 1 位表示亮灭，因此完整一帧需要：

```text
8192 bit ÷ 8 = 1024 byte
```

SSD1306 常把垂直方向的 64 行划分为 8 个页：

```text
Page 0：第 0～7 行
Page 1：第 8～15 行
...
Page 7：第 56～63 行
```

向某个页的某一列写入一个字节时，该字节的 8 个二进制位对应这一列中的 8 个垂直像素。

#### 特点

- 一页高度为 8 个像素。
- 一页包含 128 个数据字节。
- 全屏显存大小为 1024 字节。
- 8×16 字符需要占用上下相邻的两个页。
- 修改一个数据字节可能同时影响 8 个垂直像素。

#### 面试易问

**Q：为什么 128×64 单色 OLED 的一帧数据是 1024 字节？**

A：因为总像素数为 8192，每个像素只需要 1 位，所以需要 8192 位，也就是 1024 字节。

**Q：SSD1306 中“页”的含义是什么？**

A：页是显存的垂直分组方式，每一页对应 8 行像素；一个显示数据字节正好控制某一列在该页中的 8 个垂直像素。

#### 易错点

- 把页地址理解成普通像素行坐标。
- 字符上下两部分写入了同一页，导致图像叠加或缺失。
- 设置列地址时高低 4 位顺序错误。
- 字模位方向和屏幕扫描方向不一致，导致字符翻转或镜像。

---

### 3. I²C 起始、停止和数据传输

#### 原理

I²C 总线主要由两根信号线组成：

- `SCL`：串行时钟线
- `SDA`：串行数据线

总线空闲时，两根线都处于高电平。

关键时序：

```text
起始条件：SCL 为高时，SDA 从高变低
停止条件：SCL 为高时，SDA 从低变高
数据有效：SCL 为高时，SDA 应保持稳定
数据切换：通常在 SCL 为低时改变 SDA
```

发送一个字节后，第 9 个时钟用于应答位。接收方通常在该时钟期间把 SDA 拉低表示 ACK。

#### 特点

- 两根线即可连接多个设备。
- 数据通常按高位在前发送。
- 每个设备使用独立地址。
- 标准 I²C 依赖开漏输出和上拉电阻。
- 软件 I²C 便于理解协议和自由选择 GPIO。
- 硬件 I²C 占用 CPU 更少，速度和时序更稳定。

#### 面试易问

**Q：I²C 为什么使用开漏输出？**

A：开漏输出允许多个器件共同连接一根总线。任何器件都只能主动拉低，不能主动推高，从而避免一个器件输出高、另一个器件输出低时发生硬短路，并能实现线与逻辑。

**Q：I²C 的数据在什么时候有效？**

A：SCL 为高电平期间 SDA 数据必须保持稳定；SDA 通常只在 SCL 为低电平时改变，起始和停止条件除外。

**Q：软件 I²C 和硬件 I²C 有什么区别？**

A：软件 I²C 由 CPU 控制 GPIO 模拟时序，灵活且适合学习；硬件 I²C 由片上外设自动产生时序，CPU 占用更低，但初始化和异常处理更复杂。

#### 易错点

- 起始和停止条件的 SDA 跳变方向写反。
- 在 SCL 高电平期间随意改变 SDA，产生意外起始或停止条件。
- 未连接上拉电阻，导致总线无法稳定回到高电平。
- 软件 I²C 延时过短，超出从设备可接受的时序范围。
- 只产生 8 个时钟而遗漏第 9 个 ACK 时钟。

### 4. 7 位地址与 8 位地址

#### 原理

I²C 设备通常使用 7 位从机地址。发送到总线上的第一个地址字节由以下部分组成：

```text
7 位从机地址 + 1 位读写位
```

以常见 OLED 地址 `0x3C` 为例：

```text
7 位地址：0x3C
写地址字节：0x3C << 1 | 0 = 0x78
读地址字节：0x3C << 1 | 1 = 0x79
```

因此，不同资料中出现的 `0x3C` 和 `0x78` 可能指的是同一个设备，只是地址表示方式不同。

#### 特点

- `0x3C` 通常表示 7 位地址。
- `0x78` 通常表示包含写位的 8 位地址字节。
- 硬件 I²C 库函数对地址参数的要求可能不同。
- 软件 I²C 直接发送字节时通常使用移位后的地址字节。

#### 面试易问

**Q：为什么同一个 OLED 有时写地址 0x3C，有时写 0x78？**

A：`0x3C` 是 7 位从机地址；`0x78` 是把 `0x3C` 左移一位并在最低位填入写操作位 0 后得到的地址字节。

**Q：使用硬件 I²C 时应该传 0x3C 还是 0x78？**

A：必须查看所用库函数的接口定义。有的接口要求 7 位地址，有的接口要求已经左移后的地址，不能凭经验混用。

#### 易错点

- 把 7 位地址再次左移，导致实际地址错误。
- 把 `0x78` 直接当作 7 位地址传入库函数。
- 模块实际地址为 `0x3D`，代码却固定使用 `0x3C`。
- 忘记最低位是读写位，不属于从机地址本身。

### 5. 命令字节与数据字节

#### 原理

SSD1306 需要区分接收到的是控制命令还是显示数据。

软件 I²C 参考实现使用控制字节完成区分：

```text
0x00：后续字节作为命令处理
0x40：后续字节作为显示数据写入显存
```

命令用于配置：

- 显示开关
- 页地址和列地址
- 对比度
- 扫描方向
- 显示偏移
- 电荷泵
- 正常/反色显示

数据则直接改变显存中的像素位。

#### 特点

- 命令改变控制器状态。
- 数据改变屏幕显示内容。
- 两者使用同一条 I²C 总线传输。
- 控制字节决定后续负载的含义。
- 命令和数据写错通道会产生不可预测显示。

#### 面试易问

**Q：OLED_WriteCommand 和 OLED_WriteData 的区别是什么？**

A：两者发送的目标地址相同，但控制字节不同。命令函数发送 `0x00`，让 SSD1306 把后续字节解释为命令；数据函数发送 `0x40`，让后续字节写入显示 RAM。

**Q：为什么设置光标要发送命令，而显示字符要发送数据？**

A：光标位置属于控制器的寻址状态，需要命令配置；字符最终转换为像素字模，应作为显示数据写入显存。

#### 易错点

- 使用数据函数发送初始化命令。
- 使用命令函数写入字模数据。
- 每次写一个数据字节都重新起停，效率较低但逻辑正确；高速刷新时应考虑连续写入。
- 修改寻址模式后仍按原来的页寻址方式发送数据。

### 6. 字模显示原理

#### 原理

OLED 并不理解字符 `'A'`、数字 `'1'` 或字符串 `"Hello"` 的含义。程序必须先把字符转换为像素点阵数据。

例如一个 8×16 字符需要：

```text
8 列 × 16 行 = 128 bit = 16 byte
```

字模数组通常以字符 ASCII 码为索引。显示字符时：

1. 根据 ASCII 码计算字模下标。
2. 设置字符上半部分光标。
3. 写入前 8 个字节。
4. 设置字符下半部分光标。
5. 写入后 8 个字节。

#### 特点

- 字模本质是像素数据表。
- 8×16 字符占两页、八列。
- 字符显示会消耗 Flash 空间保存字模。
- 中文字符通常需要更大的 16×16 或 24×24 字模。
- 更换字体需要更换字模数组和显示算法。

#### 面试易问

**Q：OLED 为什么需要字库？**

A：OLED 控制器只接收像素数据，并不知道字符编码的图形含义。字库用于把字符编码映射成具体点阵。

**Q：一个 8×16 单色字符为什么占 16 字节？**

A：8×16 共 128 个像素，每个像素占 1 位，因此需要 128 位，也就是 16 字节。

#### 易错点

- 字模下标没有减去起始字符 `' '`，造成数组访问错误。
- 字模数组与显示函数约定的存储顺序不一致。
- 传入中文 UTF-8 字符，却使用只支持单字节 ASCII 的函数。
- 字符列坐标超出 16 列，导致内容越界或换行异常。

---

### 7. 十进制、十六进制和二进制显示

#### 原理

数字显示函数并不是直接把整数发送给 OLED，而是先逐位提取数字，再转换成字符。

十进制某一位的提取公式：

```text
当前位 = Number / 10^(剩余位数) % 10
```

十六进制使用底数 16，二进制使用底数 2。提取完成后：

- `0～9` 加 `'0'` 转成数字字符。
- 十六进制 `10～15` 转成 `A～F`。
- 带符号数先显示 `+` 或 `-`，再显示绝对值。

#### 特点

- 显示位数由 `Length` 参数控制。
- 位数不足时通常在高位补 0。
- 超出指定长度的高位会被截断。
- 整数幂和除法易理解，但效率不一定最高。
- 调试显示优先追求清晰和通用性。

#### 面试易问

**Q：OLED_ShowNum 如何逐位显示一个整数？**

A：函数从最高位开始，通过除以对应的 10 次幂并对 10 取余提取当前数字，再加 `'0'` 转换成 ASCII 字符。

**Q：如何显示十六进制数字中的 A～F？**

A：先取得 0～15 的单个十六进制位；若小于 10，加 `'0'`；否则减 10 后加 `'A'`。

#### 易错点

- `Length` 小于实际数字位数，导致高位被丢弃。
- `Length` 过大，前面出现大量补零。
- 对最小负数直接取反可能溢出。
- 二进制长度超过变量实际位宽，导致显示结果与预期不符。

### 8. OLED 初始化顺序

#### 原理

SSD1306 上电后需要通过一系列命令配置内部时钟、扫描范围、偏移、方向、对比度、电荷泵等参数。

典型顺序是：

```text
上电等待
→ 关闭显示
→ 配置时钟和多路复用率
→ 配置偏移、起始行和扫描方向
→ 配置 COM 引脚
→ 配置对比度、预充电和 VCOMH
→ 开启电荷泵
→ 开启显示
→ 清屏
```

初始化期间先关闭显示，可以避免配置尚未完成时出现闪屏或随机内容。

#### 特点

- 初始化命令与屏幕分辨率和模块接线有关。
- 扫描方向命令决定屏幕是否镜像或上下颠倒。
- 对比度影响亮度与功耗。
- 电荷泵配置错误可能导致屏幕不亮。
- 上电后应留出稳定时间再发送命令。

#### 面试易问

**Q：OLED 为什么初始化后仍要清屏？**

A：控制器的显示 RAM 上电后不一定为全 0，可能包含随机值；清屏可以保证首次显示内容干净、可预测。

**Q：屏幕显示上下颠倒应该修改什么？**

A：通常需要调整段重映射命令和 COM 扫描方向命令，例如 `0xA0/0xA1`、`0xC0/0xC8`。

#### 易错点

- 上电后立即发命令，没有等待电源稳定。
- 忘记发送开屏命令 `0xAF`。
- 忘记开启内部电荷泵。
- 将 128×32 模块的参数用于 128×64 模块。
- 初始化后不清屏，出现随机亮点。

## 本节核心记忆

```text
OLED 显示链路：
字符/数字 → 字模或数字拆分 → 显示数据 → I²C → SSD1306 显存 → 像素点亮
```

```text
I²C 关键时序：
SCL 高时 SDA 高→低：起始条件
SCL 高时 SDA 低→高：停止条件
SCL 高时：数据必须保持稳定
```

```text
常见地址：
7 位地址 0x3C
8 位写地址 0x78
```

```text
SSD1306 页结构：
128 列 × 8 页 × 每页 8 行
全屏显存 = 128 × 8 = 1024 字节
```

```text
控制字节：
0x00 → 后续是命令
0x40 → 后续是显示数据
```

## 开发过程总结

### 问题 1：OLED 完全不亮

现象：

- 程序编译和下载均正常。
- OLED 屏幕没有任何显示。
- 调用 `OLED_ShowString()` 后仍无反应。

排查过程：

1. 检查 VCC、GND、SCL、SDA 是否接错。
2. 检查 OLED 与 STM32 是否共地。
3. 检查 PB8、PB9 是否与代码定义一致。
4. 检查 GPIOB 时钟是否已经开启。
5. 检查模块地址是 `0x3C` 还是 `0x3D`。
6. 检查 SCL 和 SDA 是否存在上拉电阻。
7. 检查初始化序列中是否发送 `0x8D、0x14、0xAF`。
8. 使用逻辑分析仪观察是否存在起始条件和地址字节。

解决方案：

- 按实际接线修改 GPIO 端口和引脚宏。
- 用 I²C 地址扫描或查阅模块资料确认地址。
- 给 SCL、SDA 增加合适的上拉电阻。
- 确认初始化最后已经开启电荷泵和显示输出。
- 先只执行 `OLED_Init()` 和 `OLED_Clear()`，再逐步加入显示函数。

### 问题 2：OLED 显示乱码或字符缺失

现象：

- 屏幕能够亮。
- 字符形状不完整、上下错位或显示乱码。
- 同一个字符的上半部分和下半部分无法对齐。

排查过程：

1. 检查字模数组是否与 8×16 显示函数匹配。
2. 检查字符字模下标是否正确减去 `' '`。
3. 检查上半部分和下半部分是否写入相邻页。
4. 检查列地址是否乘以 8。
5. 检查字符是否超出字模支持的 ASCII 范围。
6. 检查扫描方向设置是否导致镜像。

解决方案：

- 使用与显示函数配套的 `OLED_F8x16` 字模。
- 对非法字符增加范围判断。
- 修正 `OLED_SetCursor()` 的页地址和列地址。
- 根据模块方向调整 `0xA0/0xA1` 与 `0xC0/0xC8`。

### 问题 3：屏幕内容左右或上下颠倒

现象：

- 字符内容正确。
- 整个画面出现镜像、上下翻转或旋转 180°。

排查过程：

1. 检查段重映射命令。
2. 检查 COM 扫描方向命令。
3. 检查模块是否采用不同的玻璃面板连接方向。
4. 检查字模数据本身是否反向。

解决方案：

- 在 `0xA0` 与 `0xA1` 之间切换列方向。
- 在 `0xC0` 与 `0xC8` 之间切换行扫描方向。
- 每次只修改一个方向命令，便于观察变化。

### 问题 4：数字显示位数不正确

现象：

- 显示结果前面有多余的 0。
- 大数字的高位被截断。
- 负数符号或绝对值异常。

排查过程：

1. 检查 `Length` 是否与期望显示位数一致。
2. 检查变量类型是否足够容纳目标数值。
3. 检查整数幂函数是否返回正确结果。
4. 检查负数绝对值计算是否发生溢出。
5. 检查显示起始列是否留出符号位置。

解决方案：

- 根据数据范围合理设置 `Length`。
- 使用 `uint32_t` 或 `int32_t` 保存较大数值。
- 带符号显示时让数值部分从 `Column + 1` 开始。
- 对最小负数使用安全的幅值转换方式。

### 问题 5：软件 I²C 偶尔通信失败

现象：

- OLED 有时正常，有时上电后不显示。
- 修改编译优化等级后现象发生变化。
- 提高主频后通信变得不稳定。

排查过程：

1. 检查短延时函数是否被编译器优化。
2. 检查 SCL、SDA 上升沿是否过慢。
3. 检查总线线长和接触是否可靠。
4. 检查上拉电阻阻值是否合适。
5. 检查发送字节后是否产生第 9 个时钟。
6. 检查是否需要真正读取 ACK 并处理异常。

解决方案：

- 使用 `volatile` 变量或定时器实现稳定延时。
- 适当增大软件 I²C 延时。
- 缩短杜邦线并改善接触。
- 根据总线电容调整上拉电阻。
- 在正式项目中增加 ACK 检测、超时和错误恢复机制。

## 本节小结

本节完成了 STM32F103C8T6 对 SSD1306 OLED 显示屏的基础驱动，理解了从 GPIO 模拟 I²C 到字符、数字最终显示在屏幕上的完整链路。

本节最重要的知识点是：

```text
OLED 不直接理解字符和数字，只理解命令与像素数据
```

```text
软件 I²C 的本质是按照协议要求控制 SCL 和 SDA 的电平变化
```

```text
SSD1306 的 128×64 显存通常按 8 页组织，每个数据字节控制 8 个垂直像素
```

```text
字符显示 = 字符编码 → 查询字模 → 设置光标 → 写入上下两页字模数据
```

OLED 是嵌入式学习中非常实用的调试工具。后续学习外部中断、定时器、PWM、输入捕获、ADC、串口和传感器时，都可以把关键变量、计数值和状态信息实时显示在 OLED 上，从而提高调试效率。
