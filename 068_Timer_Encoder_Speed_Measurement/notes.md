# 18 — TIM编码器接口：编码器接口测速

> 整理说明：本节对应江协科技《STM32入门教程-2023版》的 **[6-8] 编码器接口测速**。由于无法直接取得视频完整字幕与逐帧代码，本文依据分集主题、课程上下文、STM32F103 官方资料及标准外设库的典型写法整理；“核心代码”属于与课程思路一致的参考实现，并非视频源码逐字转录。

## 实验概述

本实验基于 STM32F103C8T6 标准外设库，将增量式编码器的 A、B 两相正交信号接入 TIM3 的 CH1 和 CH2，使用定时器硬件编码器接口自动完成：

1. 识别编码器脉冲边沿
2. 判断旋转方向
3. 控制 CNT 自动加计数或减计数
4. 每隔固定时间读取一次计数增量
5. 将计数增量作为带方向的速度值显示在 OLED 上

本节的核心目标是掌握：

> A/B 相正交信号 → TIM3 编码器接口硬件计数 → TIM2 固定闸门采样 → 读取并清零 CNT → 得到带方向速度

与使用 EXTI 中断逐个处理编码器脉冲相比，定时器编码器接口把频繁的加减计数工作交给硬件完成，可以显著减少 CPU 中断负担，尤其适合电机测速、位置反馈和后续 PID 闭环控制。

## 硬件连接

本实验默认使用 TIM3 的默认映射引脚 PA6、PA7。具体电平、供电电压和上拉方式应以编码器模块原理图为准。

| 器件 / 信号 | STM32 引脚 | 外设功能 | 说明 |
|---|---:|---|---|
| 编码器 A 相 | PA6 | TIM3_CH1 | 接收第一路正交脉冲 |
| 编码器 B 相 | PA7 | TIM3_CH2 | 接收第二路正交脉冲 |
| 编码器 VCC | 3.3V | 电源 | 优先确认模块是否兼容 3.3V |
| 编码器 GND | GND | 地 | 必须与 STM32 共地 |
| OLED SCL | PB8 | 软件 I²C 时钟 | 沿用前面 OLED 实验的连接 |
| OLED SDA | PB9 | 软件 I²C 数据 | 沿用前面 OLED 实验的连接 |
| ST-Link V2 | PA13、PA14、GND、3.3V | SWD | 用于下载和调试 |

> 注意 1：PA6 和 PA7 分别对应 TIM3_CH1、TIM3_CH2，不需要额外配置 AFIO 重映射。

> 注意 2：如果显示的正负方向与预期相反，可以交换 A、B 两相信号线，或者反转其中一路输入极性。

> 注意 3：机械旋转编码器通常需要较强数字滤波；高速电机编码器则需要根据最高脉冲频率降低滤波强度，避免有效边沿被滤除。

### 2. Encoder.c

```c
#include "stm32f10x.h"                             // 引入 STM32F10x 标准外设库的寄存器、结构体和函数声明
#include "Encoder.h"                               // 引入本模块函数声明，保证定义与声明保持一致

void Encoder_Init(void)                            // 定义编码器接口初始化函数
{                                                  // 进入编码器初始化函数体
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM3, ENABLE); // 开启 TIM3 外设时钟，使定时器寄存器可以被正常访问
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE); // 开启 GPIOA 外设时钟，使 PA6 和 PA7 可以被配置

    GPIO_InitTypeDef GPIO_InitStructure;           // 定义 GPIO 初始化结构体，用于集中保存输入引脚配置
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPU;  // 将 PA6、PA7 配置为上拉输入，防止开路时输入电平悬空
    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_6 | GPIO_Pin_7; // 同时选择 PA6 与 PA7，分别接收编码器 A 相和 B 相
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz; // 填写 GPIO 速度字段；输入模式下该字段不决定采样速度
    GPIO_Init(GPIOA, &GPIO_InitStructure);          // 将上述配置写入 GPIOA 控制寄存器

    TIM_TimeBaseInitTypeDef TIM_TimeBaseInitStructure; // 定义 TIM3 时基初始化结构体，用于配置计数器基本参数
    TIM_TimeBaseInitStructure.TIM_ClockDivision = TIM_CKD_DIV1; // 设置数字滤波采样时钟分频系数为 1
    TIM_TimeBaseInitStructure.TIM_CounterMode = TIM_CounterMode_Up; // 填写默认向上计数模式；编码器接口会动态控制实际方向
    TIM_TimeBaseInitStructure.TIM_Period = 65536 - 1; // 将 ARR 设为 65535，充分利用 16 位计数器的完整范围
    TIM_TimeBaseInitStructure.TIM_Prescaler = 1 - 1; // 将 PSC 设为 0，避免对定时器内部时钟进行预分频
    TIM_TimeBaseInitStructure.TIM_RepetitionCounter = 0; // 重复计数器设为 0；TIM3 为通用定时器，此字段不参与本实验
    TIM_TimeBaseInit(TIM3, &TIM_TimeBaseInitStructure); // 把时基参数写入 TIM3 的 PSC、ARR、CR1 等寄存器

    TIM_ICInitTypeDef TIM_ICInitStructure;         // 定义输入捕获初始化结构体，配置两路输入滤波和极性
    TIM_ICInitStructure.TIM_Channel = TIM_Channel_1; // 先选择 TIM3 通道 1，对应编码器 A 相输入
    TIM_ICInitStructure.TIM_ICFilter = 0x0F;       // 设置较强数字滤波，抑制机械触点抖动和短时干扰脉冲
    TIM_ICInitStructure.TIM_ICPolarity = TIM_ICPolarity_Rising; // 选择不反相极性；编码器模式仍会根据正交边沿计数
    TIM_ICInitStructure.TIM_ICPrescaler = TIM_ICPSC_DIV1; // 设置输入捕获预分频为 1，每个有效输入事件都参与处理
    TIM_ICInitStructure.TIM_ICSelection = TIM_ICSelection_DirectTI; // 将通道 1 连接到自身直接输入 TI1
    TIM_ICInit(TIM3, &TIM_ICInitStructure);        // 将通道 1 的输入滤波、极性等参数写入 TIM3

    TIM_ICInitStructure.TIM_Channel = TIM_Channel_2; // 改为选择 TIM3 通道 2，对应编码器 B 相输入
    TIM_ICInitStructure.TIM_ICFilter = 0x0F;       // 为 B 相设置与 A 相一致的数字滤波强度
    TIM_ICInitStructure.TIM_ICPolarity = TIM_ICPolarity_Rising; // 保持 B 相输入不反相，便于按默认方向判定
    TIM_ICInitStructure.TIM_ICPrescaler = TIM_ICPSC_DIV1; // 设置通道 2 不进行输入事件分频
    TIM_ICInitStructure.TIM_ICSelection = TIM_ICSelection_DirectTI; // 将通道 2 连接到自身直接输入 TI2
    TIM_ICInit(TIM3, &TIM_ICInitStructure);        // 将通道 2 的输入滤波、极性等参数写入 TIM3

    TIM_EncoderInterfaceConfig(TIM3, TIM_EncoderMode_TI12, TIM_ICPolarity_Rising, TIM_ICPolarity_Rising); // 启用 TI1 和 TI2 双通道编码器模式并保持两路输入不反相
    TIM_SetCounter(TIM3, 0);                       // 在启动计数前把 CNT 清零，建立统一的初始位置和测速基准
    TIM_Cmd(TIM3, ENABLE);                         // 使能 TIM3，从此硬件根据 A/B 相状态自动控制 CNT 加减
}                                                  // 结束编码器初始化函数

int16_t Encoder_Get(void)                          // 定义读取一个闸门周期内编码器增量的函数
{                                                  // 进入编码器增量读取函数体
    int16_t Temp;                                  // 定义 16 位有符号临时变量，用于保留方向信息
    Temp = (int16_t)TIM_GetCounter(TIM3);          // 读取 TIM3 CNT，并按补码解释为正数或负数
    TIM_SetCounter(TIM3, 0);                       // 读取后立即清零 CNT，使下一测速周期重新累计增量
    return Temp;                                   // 返回本周期计数值；正负号表示方向，绝对值反映快慢
}                                                  // 结束编码器增量读取函数
```

### 4. Timer.c

```c
#include "stm32f10x.h"                             // 引入 STM32F10x 标准外设库定义
#include "Timer.h"                                 // 引入本模块公开函数声明

void Timer_Init(void)                              // 定义 TIM2 定时中断初始化函数
{                                                  // 进入 TIM2 初始化函数体
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM2, ENABLE); // 开启 TIM2 时钟，使其能够产生周期性更新事件
    TIM_InternalClockConfig(TIM2);                 // 选择 TIM2 内部时钟作为计数时钟来源

    TIM_TimeBaseInitTypeDef TIM_TimeBaseInitStructure; // 定义 TIM2 时基初始化结构体
    TIM_TimeBaseInitStructure.TIM_ClockDivision = TIM_CKD_DIV1; // 设置时钟分频参数为 1
    TIM_TimeBaseInitStructure.TIM_CounterMode = TIM_CounterMode_Up; // 配置 TIM2 从 0 向上计数
    TIM_TimeBaseInitStructure.TIM_Period = 10000 - 1; // 设置 ARR 为 9999，使计数器累计 10000 次后更新
    TIM_TimeBaseInitStructure.TIM_Prescaler = 7200 - 1; // 将 72 MHz 定时器时钟分频到 10 kHz
    TIM_TimeBaseInitStructure.TIM_RepetitionCounter = 0; // 重复计数器设为 0，通用定时器中不参与本实验
    TIM_TimeBaseInit(TIM2, &TIM_TimeBaseInitStructure); // 写入 TIM2 时基参数，得到约 1 秒更新周期

    TIM_ClearFlag(TIM2, TIM_FLAG_Update);          // 清除初始化过程中可能产生的更新标志，防止启动后立即误进中断
    TIM_ITConfig(TIM2, TIM_IT_Update, ENABLE);     // 允许 TIM2 更新事件向 NVIC 发出中断请求

    NVIC_PriorityGroupConfig(NVIC_PriorityGroup_2); // 设置 NVIC 优先级分组为 2 位抢占优先级和 2 位响应优先级
    NVIC_InitTypeDef NVIC_InitStructure;           // 定义 NVIC 初始化结构体
    NVIC_InitStructure.NVIC_IRQChannel = TIM2_IRQn; // 选择 TIM2 全局中断通道
    NVIC_InitStructure.NVIC_IRQChannelCmd = ENABLE; // 使能 TIM2 中断通道
    NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 2; // 设置 TIM2 的抢占优先级为 2
    NVIC_InitStructure.NVIC_IRQChannelSubPriority = 1; // 设置 TIM2 的响应优先级为 1
    NVIC_Init(&NVIC_InitStructure);                // 将中断优先级配置写入 NVIC

    TIM_Cmd(TIM2, ENABLE);                         // 启动 TIM2，开始形成固定 1 秒测速闸门
}                                                  // 结束 TIM2 初始化函数
```

### 5. main.c

```c
#include "stm32f10x.h"                             // 引入 STM32F10x 芯片定义和标准外设库接口
#include "OLED.h"                                  // 引入 OLED 显示驱动接口
#include "Encoder.h"                               // 引入 TIM3 编码器接口驱动
#include "Timer.h"                                 // 引入 TIM2 固定闸门定时器驱动

volatile int16_t Speed;                            // 定义带方向的速度计数值，并用 volatile 防止被编译器错误优化

int main(void)                                     // 定义程序入口函数
{                                                  // 进入主函数
    OLED_Init();                                   // 初始化 OLED，准备显示测速结果
    Encoder_Init();                                // 初始化 TIM3 编码器接口，使硬件开始接收 A/B 相脉冲
    Timer_Init();                                  // 初始化 TIM2 的 1 秒定时中断，周期性读取编码器增量

    OLED_ShowString(1, 1, "Speed:");               // 在 OLED 第一行显示速度标签，便于观察数据含义

    while (1)                                      // 进入无限循环，使程序持续显示最新测速结果
    {                                              // 进入主循环体
        OLED_ShowSignedNum(1, 7, Speed, 5);        // 以有符号十进制显示速度计数，正负号表示旋转方向
    }                                              // 结束本次主循环并立即开始下一次刷新
}                                                  // 结束主函数；嵌入式程序正常情况下不会运行到此处

void TIM2_IRQHandler(void)                         // 定义 TIM2 更新中断服务函数，每个测速闸门结束时执行一次
{                                                  // 进入 TIM2 中断服务函数
    if (TIM_GetITStatus(TIM2, TIM_IT_Update) == SET) // 判断本次中断是否由 TIM2 更新事件触发
    {                                              // 进入更新中断处理分支
        Speed = Encoder_Get();                     // 读取 TIM3 的带方向增量并清零 CNT，形成新速度采样值
        TIM_ClearITPendingBit(TIM2, TIM_IT_Update); // 清除 TIM2 更新中断挂起标志，允许下一周期再次触发
    }                                              // 结束更新事件判断分支
}                                                  // 结束 TIM2 中断服务函数
```

> 说明：上面的 `Speed` 表示 **1 秒闸门内的编码器计数增量**，单位是“计数/秒”，并不天然等于 RPM。只有知道编码器每转对应的有效计数数 `N` 后，才能换算机械转速：

```text
RPM = Speed × 60 / N
```

若测速闸门不是 1 秒，而是 `Tg` 秒，则：

```text
RPM = Speed × 60 / (N × Tg)
```

## 关键知识点

### 1. 增量式正交编码器

#### 原理

增量式编码器通常输出 A、B 两路周期相同、相位相差约 90° 的数字信号。

当编码器沿一个方向旋转时，A 相领先 B 相；反向旋转时，B 相领先 A 相。控制器不只统计脉冲数量，还可以根据两相信号的先后关系判断方向。

典型状态序列可以表示为：

```text
正转：00 → 01 → 11 → 10 → 00
反转：00 → 10 → 11 → 01 → 00
```

#### 特点

- 可以同时获得位移增量和运动方向。
- 两相信号具有一定抗干扰能力，非法状态跳变可被识别或滤除。
- 输出只有相对位置，系统掉电后通常无法保留绝对位置。
- 常见于旋转旋钮、电机编码器、光栅尺和运动控制系统。

#### 面试易问

**Q：为什么编码器需要 A、B 两相信号？**

A：单路脉冲只能反映发生了多少次变化，无法判断运动方向；A、B 两相存在固定相位差，比较谁领先即可判断正反转。

**Q：增量式编码器和绝对式编码器有什么区别？**

A：增量式编码器输出相对位移脉冲，重新上电后通常需要重新寻找零点；绝对式编码器的每个位置对应唯一编码，上电即可知道当前位置。

#### 易错点

- 只连接 A 相，导致只能计数，无法可靠判断方向。
- A、B 相接反后，正负方向与预期相反。
- 忽略编码器模块的输出电压，直接把 5V 非兼容信号接入 MCU。
- 把机械编码器每个“卡点”误认为固定只产生一个计数。

### 2. TIM 编码器接口

#### 原理

STM32 通用定时器和高级定时器中的编码器接口复用输入捕获通道 1、通道 2：

```text
编码器 A 相 → TI1 → 滤波与极性选择 ┐
                                      ├→ 编码器接口 → 控制 CNT 时钟与方向
编码器 B 相 → TI2 → 滤波与极性选择 ┘
```

编码器接口根据两路信号的电平与边沿关系，自动决定：

- 当前是否产生一次有效计数
- CNT 应加 1 还是减 1
- 当前旋转方向是什么

CPU 不需要在每个脉冲到来时进入中断。

#### 特点

- 计数和判向由硬件完成，CPU 占用低。
- 可使用输入数字滤波抑制毛刺。
- 可选择 TI1、TI2 或 TI12 编码器模式。
- 一个定时器进入编码器模式后，相关通道和计数器资源会被占用。

#### 面试易问

**Q：编码器接口与 EXTI 编码器计数相比有什么优势？**

A：EXTI 方案需要 CPU 在每个边沿进入中断并执行判向、加减计数；定时器编码器接口由硬件自动完成这些工作，适合高脉冲频率和实时性要求较高的场景。

**Q：`TIM_EncoderMode_TI12` 表示什么？**

A：表示计数器同时依据 TI1 和 TI2 两路输入的有效边沿进行正交解码，能够获得更高的有效计数分辨率。

#### 易错点

- 把编码器接到不是该定时器 CH1、CH2 的引脚。
- 配置输入捕获后忘记调用 `TIM_EncoderInterfaceConfig()`。
- 在调用编码器接口配置后又重新初始化输入捕获，从而覆盖极性等配置。
- 认为编码器模式仍由 `TIM_CounterMode_Up` 固定决定方向。

### 3. 固定闸门时间测速

#### 原理

本实验采用测频法，也可称为 M 法测速：

1. 让编码器接口在固定时间 `Tg` 内累计计数。
2. 闸门结束时读取 `CNT`。
3. 将 `CNT` 清零。
4. 读取值就是该周期内的带方向计数增量。

若每转有效计数为 `N`，则：

```text
转速（RPM） = ΔCNT × 60 / (N × Tg)
```

#### 特点

- 实现简单，适合中高速测量。
- 闸门时间固定，计算量小。
- 闸门越长，低速分辨率越高。
- 闸门越短，速度响应越快。

#### 面试易问

**Q：为什么读取 CNT 后要清零？**

A：清零后，下一次读取到的值才是新的固定时间窗口内的增量；如果不清零，读到的是累计位置，需要再与上一次结果做差。

**Q：闸门时间应该如何选择？**

A：要在速度响应和低速分辨率之间折中。控制系统常使用较短周期以提高响应速度；低速显示可使用较长周期或结合测周法。

#### 易错点

- 把固定周期内的脉冲数直接当成 RPM。
- 修改 TIM2 周期后没有同步修改转速换算公式。
- 闸门过短导致低速时结果在 0 和 1 之间跳变。
- 闸门过长导致速度变化显示明显滞后。

### 4. 16 位计数器与补码

#### 原理

TIM3 的 CNT 是 16 位无符号寄存器，范围为：

```text
0 ~ 65535
```

反向计数经过 0 后会变成 65535、65534……。将该 16 位数据强制转换为 `int16_t` 后，依据补码规则：

```text
65535 → -1
65534 → -2
32768 → -32768
```

因此，只要一个测速周期内的净增量保持在 `-32768 ~ 32767`，就可以直接得到有符号结果。

#### 特点

- 无需额外判断上溢或下溢即可表示小范围正负增量。
- 代码简洁，非常适合周期性读取后清零的测速方式。
- 可自然保留旋转方向。
- 有效无歧义范围只有 16 位有符号范围。

#### 面试易问

**Q：为什么 `TIM_GetCounter()` 返回无符号数，却能得到负速度？**

A：CNT 中保存的是 16 位二进制值，将其转换成 `int16_t` 后会按补码解释，高位为 1 的数被解释为负数。

**Q：这种写法什么时候会失效？**

A：当单个采样周期内净计数绝对值超过 32767 时会发生符号混淆，需要缩短采样周期、使用 32 位定时器或做软件溢出扩展。

#### 易错点

- 使用 `uint16_t` 保存速度，导致反转时显示成很大的正数。
- 认为 ARR 设为 65535 后就永远不会有溢出问题。
- 采样周期过长或转速过高，造成超过有符号安全范围。
- 在不同位宽整数之间转换时忽略符号扩展。

---

### 5. 编码器输入数字滤波

#### 原理

TIM 输入通道的数字滤波器会对输入进行连续采样，只有信号在一定时间内保持稳定，才把它识别为有效跳变。

滤波越强：

- 能滤除的窄毛刺越宽
- 输入确认延迟越大
- 可接受的最高脉冲频率越低

本实验使用 `0x0F`，适合机械编码器教学实验，但不一定适合所有高速电机编码器。

#### 特点

- 由硬件完成，不需要软件延时消抖。
- 能同时作用于 A、B 两路输入。
- 可提高静止和低速时计数稳定性。
- 参数必须根据噪声和最高输入频率折中。

#### 面试易问

**Q：滤波值是不是越大越好？**

A：不是。滤波越强虽然抗干扰越好，但也会降低可识别的最高边沿频率，并增加信号确认延迟。

**Q：高速电机编码器为什么可能丢计数？**

A：如果数字滤波过强、输入电平质量差或脉冲频率超过定时器输入能力，有效边沿可能无法被识别。

#### 易错点

- 机械编码器不加滤波，静止时速度仍随机跳变。
- 高速编码器仍使用最大滤波，造成高速丢脉冲。
- A、B 两路滤波参数不一致，影响判向稳定性。
- 只用软件延时消抖，阻塞主程序并遗漏快速边沿。

---

### 6. 计数分辨率与每转脉冲数

#### 原理

编码器规格常写成 PPR、CPR 或每转线数，但不同厂商对术语定义可能不同。

在双通道四倍频解码时，一个完整 A/B 周期最多可贡献 4 个计数，因此有效每转计数可能为：

```text
N = 编码器周期数/转 × 4 × 减速比
```

机械旋钮还可能存在“每个卡点对应 2 个或 4 个边沿”等结构差异，必须实测确认。

#### 特点

- `N` 决定位置分辨率和 RPM 换算比例。
- 分辨率越高，同一转速下计数越多。
- 电机减速器会改变输出轴每转对应的编码器计数。
- 实际工程应根据数据手册和实测校准。

#### 面试易问

**Q：编码器标称 500 PPR，为什么程序一圈可能读到 2000？**

A：如果 500 PPR 表示每相每转 500 个完整周期，而定时器使用 A/B 双通道四倍频解码，则每转有效计数可能是 500 × 4 = 2000。

**Q：如何确认实际每转计数？**

A：将计数器清零，手动让机械轴精确旋转一圈，读取累计计数，多次测量并结合减速比校验。

#### 易错点

- 混淆 PPR、CPR、线数和四倍频后的有效计数。
- 忽略减速电机的齿轮减速比。
- 直接套用网络上的每转计数参数。
- 未区分电机轴编码器计数和减速箱输出轴计数。

---

### 7. 速度变量的中断共享

#### 原理

`Speed` 在 TIM2 中断中被写入，在主循环中被读取。编译器无法自动理解中断会异步修改变量，因此需要使用 `volatile`：

```text
volatile int16_t Speed;
```

这会要求编译器每次使用时都从内存重新读取，而不是长期缓存旧值。

#### 特点

- 适合中断与主循环共享的简单状态变量。
- 16 位对齐读写在 Cortex-M3 上通常可一次完成。
- `volatile` 解决可见性问题，不等于线程安全或原子性保证。
- 复杂多字节数据仍可能需要临界区或双缓冲。

#### 面试易问

**Q：`volatile` 能保证变量操作原子性吗？**

A：不能。`volatile` 只要求每次真实访问内存，不能防止复合操作被中断打断，也不能替代互斥、临界区或内存屏障。

**Q：为什么普通全局变量可能显示不更新？**

A：在优化编译时，主循环可能把变量值缓存在寄存器中；如果没有 `volatile`，编译器不知道中断会修改它。

#### 易错点

- 中断共享变量忘记加 `volatile`。
- 认为加了 `volatile` 后所有并发访问都安全。
- 在中断里执行 OLED 刷屏等耗时操作。
- 中断处理函数没有及时清除挂起标志。

---

### 8. 编码器接口与电机闭环控制

#### 原理

编码器测速常与 PWM 电机驱动和 PID 控制组合：

```text
目标速度 → PID 控制器 → PWM 占空比 → 电机
                 ↑                 ↓
                 └── 编码器实际速度 ┘
```

编码器提供实际速度反馈，控制器根据目标值与实际值之差不断修正 PWM，占空比最终稳定在维持目标速度所需的水平。

#### 特点

- 是速度闭环和位置闭环的基础反馈环节。
- 采样周期应稳定，否则会影响控制器参数和动态性能。
- 测速噪声会直接影响 PID 输出。
- 实际项目需要同时考虑方向、饱和、死区和机械惯性。

#### 面试易问

**Q：为什么 PID 控制要求测速周期固定？**

A：离散 PID 的积分和微分项都与采样时间有关；周期变化会改变控制器等效参数，引起响应不一致甚至振荡。

**Q：编码器测速值跳动时可以怎样处理？**

A：可以优化硬件滤波、调整输入数字滤波、缩短或延长闸门、使用滑动平均或低通滤波，并检查机械同轴度和供电干扰。

#### 易错点

- 在主循环中用不固定延时读取速度，却按固定周期设计 PID。
- 直接对强噪声速度做微分，导致控制输出剧烈抖动。
- 未限制 PWM 输出范围，造成积分饱和。
- 方向符号与电机控制方向定义不一致。

---

## 本节核心记忆

```text
编码器接口 = CH1/CH2 正交输入 + 硬件判向 + CNT 自动加减
```

```text
测速 = 固定时间读取 CNT + 读取后清零
```

```text
Speed 的正负表示方向，绝对值表示一个闸门周期内的计数快慢
```

```text
RPM = ΔCNT × 60 /（每转有效计数 × 闸门时间）
```

```text
机械编码器重视滤波；高速电机编码器重视最高输入频率与溢出范围
```

## 开发过程总结

### 问题 1：旋转编码器后，Speed 始终为 0

现象：

- OLED 能正常显示
- 正反旋转编码器时数字不变化
- TIM2 定时中断可能仍在正常运行

排查过程：

1. 检查编码器 A、B 相是否接到 PA6、PA7
2. 检查 TIM3 和 GPIOA 时钟是否开启
3. 检查 PA6、PA7 是否被其他外设占用
4. 检查编码器模块是否供电并与 STM32 共地
5. 在调试器中观察 TIM3->CNT 是否随旋转变化
6. 检查是否调用了 `TIM_Cmd(TIM3, ENABLE)`

解决方案：

- 按 TIM3 默认映射重新连接 A/B 相
- 确认 `Encoder_Init()` 在 `Timer_Init()` 之前执行
- 移除与 SPI1、ADC 或其他 PA6/PA7 功能的冲突配置
- 根据模块输出类型选择上拉输入、浮空输入或外部上拉

### 问题 2：正转显示负数，反转显示正数

现象：

- 计数绝对值基本正常
- 正负方向与预期完全相反

排查过程：

1. 确认软件对“正方向”的定义
2. 检查 A、B 相是否接反
3. 检查 `TIM_EncoderInterfaceConfig()` 的输入极性设置
4. 检查机械安装方向是否与程序坐标系一致

解决方案：

- 最直接的方法是交换 A、B 两根信号线
- 或将其中一路极性改为 `TIM_ICPolarity_Falling`
- 也可以在应用层对结果取反，但应统一整个系统的方向定义

### 问题 3：编码器静止时，Speed 仍随机跳动

现象：

- 编码器未旋转
- OLED 偶尔显示 `1`、`-1` 或更大的随机值

排查过程：

1. 检查输入引脚是否悬空
2. 检查编码器模块是否有可靠上拉
3. 检查信号线是否过长或靠近电机电源线
4. 检查 TIM 输入滤波是否配置
5. 检查电源去耦和地线连接

解决方案：

- 使用上拉输入或增加合适的外部上拉电阻
- 增大 `TIM_ICFilter` 以抑制机械抖动
- 缩短信号线并改善布线、屏蔽和共地
- 在编码器电源附近增加去耦电容

### 问题 4：低速时速度显示在 0 和 1 之间跳变

现象：

- 快速旋转时显示较稳定
- 极低速度时读数量化明显

排查过程：

1. 检查测速闸门是否过短
2. 确认每转有效计数是否较低
3. 判断应用更关心响应速度还是低速分辨率
4. 观察多周期累计后的稳定程度

解决方案：

- 适当延长测速闸门
- 对多个采样值做滑动平均
- 低速区切换到测周法
- 使用更高分辨率编码器

### 问题 5：高速旋转时读数突然变负或明显异常

现象：

- 中低速时结果正常
- 高速时数值跳变、符号翻转或变小

排查过程：

1. 计算一个闸门周期内最大可能计数
2. 检查是否超过 `int16_t` 的 32767
3. 检查输入数字滤波是否过强
4. 检查信号上升沿质量和输入频率
5. 检查每转计数和减速比是否计算正确

解决方案：

- 缩短 TIM2 测速周期
- 使用软件溢出扩展或 32 位定时器
- 降低输入滤波强度
- 使用施密特整形、差分接收或更可靠的编码器接口电路

### 问题 6：换算出的 RPM 与实际差很多

现象：

- 原始 Speed 变化合理
- RPM 比真实转速高 2 倍、4 倍或更多

排查过程：

1. 查清编码器规格中的 PPR/CPR 定义
2. 确认 TIM3 是否使用 TI12 双通道解码
3. 测量机械轴完整旋转一圈的实际累计计数
4. 检查是否存在减速箱
5. 检查闸门时间是否确实为 1 秒

解决方案：

- 使用实测的“每转有效计数”参与公式
- 把减速比计入换算
- 根据实际 TIM2 周期修改 `Tg`
- 不要把编码器标称线数直接等同于四倍频后的计数数

## 结果展示

> 实验 1：烧录程序后，旋转编码器时 TIM3 的 CNT 能够由硬件自动增加或减少。✅

> 实验 2：每隔约 1 秒，TIM2 中断读取并清零 TIM3 计数器，OLED 显示该周期内的带方向计数增量。✅

> 实验 3：改变旋转方向后，OLED 显示值的正负号随之改变；提高旋转速度后，计数绝对值增大。✅

> 实验 4：编码器停止旋转后，下一个完整测速周期的 Speed 回到 0。✅

## 本节小结

本节通过 TIM3 编码器接口完成了增量式编码器测速，掌握了 A/B 相正交信号、硬件判向、CNT 加减计数和固定闸门采样的完整流程。

最重要的开发思路是：

```text
TIM3 负责“高速、频繁、简单”的脉冲计数
TIM2 负责“固定周期”的速度采样
主循环只负责显示和后续控制
```

相比 EXTI 软件计数，定时器编码器接口能够降低 CPU 负担，并且天然保留方向信息。继续向后扩展时，可以把原始计数增量换算成 RPM，再与 PWM 电机驱动、PID 速度环和位置环结合，形成完整的闭环运动控制系统。
