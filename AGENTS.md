# Codex 工作约定

本文档定义了在处理本仓库内容时的输出规范与协作方式。

## 仓库目标

为江协科技 STM32 标准库系列课程的每一节实验建立结构化学习档案，内容包含硬件连接、核心代码解析、关键知识点和开发总结。最终目标：
- 供用户（27届应届硕士研究生）秋招复习时快速回溯
- 可作为面试作品集展示

## 文档结构（每章必须遵循）

每节课输出一个章节文件夹，命名为 `XX-课程英文名称/`，内含：

```
XX-CourseName/
├── README.md       ← 完整实验报告（按下方模板）
├── Code/           ← 核心代码文件
├── Hardware/       ← 硬件连接照片
└── Results/        ← 实验结果截图/视频链接
```

### README.md 内容模板

每个 `README.md` 必须严格按以下结构编写：

```
## 📌 实验概述
简要描述本节实验做了什么事，实现什么功能。

## 🔌 硬件连接
列出使用的外设、引脚对应关系，必要时说明接线方式。

## 🧩 核心代码
展示核心代码段，每段后附一两句解释，说明这段代码的作用和关键点。

## 💡 关键知识点
整理本节涉及的核心概念、寄存器/外设原理、易错点。
用列表形式，每条自包含，方便复习时扫读。

## 🧠 开发过程总结
记录实际开发中遇到的问题、解决思路、个人理解深化。
⚠️ 一定要记录踩坑和调试过程——这是面试中最有价值的素材。

## 📎 结果展示
- 图片：使用 `![描述](Hardware/xxx.jpg)` 相对路径
- 视频：提供 B站未列出/仅链接可见的视频地址
```

## 输出风格约定

- 中文为主，术语首次出现时附带英文（如 GPIO (General Purpose Input Output)）
- 代码解释要直击关键，不做逐行翻译式注释
- 知识点要面向**面试**，标注可能被问到的问题（"面试易问"）
- 踩坑经历要写成"问题 → 排查 → 解决"三段式
- 整体语气：像一位有经验的学长在帮你整理笔记，专业但不刻板

## 协作流程（每节课一个循环）

1. **用户做实验** → 拍硬件连接图、录结果视频
2. **用户存放素材**：
   - 硬件连接图 → `D:\01_STM32实验记录\STM32实验硬件连接图\`
   - 实验结果视频 → `D:\01_STM32实验记录\STM32实验结果视频\`
3. **用户通知我**，提供：
   - 本章名称（如 "跑马灯实验"）
   - 江协科技对应视频的 B站链接
4. **我来完成**：
   - 访问视频链接提取内容
   - 创建章节文件夹和 Code/main.c（核心代码）
   - 撰写完整 README.md（关键知识 + 开发总结）
5. **用户审核文档**
6. **用户将图片/视频从 D 盘复制到对应章节的 Hardware/ 和 Results/ 目录**
7. **用户推送至 GitHub**

## 更新规范

- 不修改已生成的过往章节，除非用户主动要求修订
- 如果后续课程深化了对前面某个知识点的理解，可以在新章节中"关联回顾"，引回前面的章节号
